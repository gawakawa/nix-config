# CLAUDE.md

Guidance for Claude Code in this repository.

## Critical constraints

- **System rebuilds need sudo — Claude must NOT run them.** The user runs these manually after config changes:
  - NixOS: `sudo nixos-rebuild switch --flake ".#nixos" --accept-flake-config --impure`
  - Darwin: `sudo darwin-rebuild switch --flake ".#mac"`
- **`hardware-configuration.nix`** is auto-generated — never edit it (statix/deadnix ignore it too).
- **Neovim** is a separate flake at `./nvim/`; run via `nix run ~/.config/nix-config/nvim --` (alias `v`).
- **`~/.claude/` (global Claude Code config) is generated by Nix from `profiles/home/claude/`.** To change the global `CLAUDE.md`, `settings.json`, skills, agents, or scripts, edit the sources under `profiles/home/claude/` — NOT `~/.claude/` directly (it is overwritten on rebuild).
- New files are invisible to flake eval until `git add`-ed.

## Commands Claude can run

- Format: `nix fmt` · Check: `nix flake check` (includes treefmt format check)
- Pre-commit hooks auto-load via direnv; defined in `flakes/pre-commit.nix`.

## Layout

flake-parts config for NixOS (x86_64-linux) + Darwin (aarch64-darwin).

- `flakes/` — flake-parts modules. Host defs in `flakes/hosts.nix`; `mkNixos`/`mkDarwin` helpers in `flakes/lib/hosts.nix`.
- `hosts/`, `home/` — per-host system / Home Manager entry points (`mac`, `nixos`).
- `profiles/` — shared modules (`home/`, `hosts/`). Shell aliases/functions live in `profiles/home/zsh/`.
- CI (`.github/workflows/ci.yml`): `nix flake check` + build both configs, cached via Cachix (`gawakawa`).
